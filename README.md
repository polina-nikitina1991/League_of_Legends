
# 📊 League of Legends ETL & Analytics Project

## 📌 Описание проекта

Аналитический проект, в котором реализован полный **ETL-пайплайн** для сбора, обработки и анализа данных матчей League of Legends с использованием Riot API.

Проект включает:

* сбор данных о матчах и игроках
* трансформацию в аналитические таблицы
* загрузку в PostgreSQL
* построение витрин (SQL views) для BI-инструментов
* подготовку данных для визуализации (DataLens)

---

## 🏗 Архитектура проекта

Проект построен по классической схеме:

```text
EXTRACT → TRANSFORM → LOAD → MART (SQL views) → DASHBOARD
```

---

## 📂 Структура репозитория

Рекомендуемая структура:

```text
lol_etl/
│
├── etl/

│   ├── analysis.sql        # аналитические запросы
│

├── main.py                 # точка входа (ETL pipeline)
├── requirements.txt        # зависимости
│
├── data/                   # (опционально) локальные данные
├── README.md
```

## 🔄 ETL пайплайн

### 1. Extract

* Получение топ-игроков (Challenger)
* Загрузка матчей по puuid
* Получение данных о предметах

### 2. Transform

Формируются таблицы:

* `players` — список игроков
* `matches` — информация о матчах
* `participants` — игроки в матчах (основная таблица)
* `match_items` — предметы игроков
* `items` — справочник предметов

### 3. Load

* Сохранение в CSV
* Загрузка в PostgreSQL

---

## 🧠 Аналитический слой (SQL)

Созданы витрины:

* `mart_champion_stats`
* `mart_role_stats`
* `mart_economy_stats`
* `mart_item_stats`
* `mart_item_champion`
* `mart_object_impact`
* `mart_vision_stats`
* `mart_cs_impact`

📌 Важно:
Витрины не соединяются между собой напрямую — используются отдельно в BI.

## 🚀 Запуск проекта

```bash
python main.py --players 5 --matches 10 --save-csv yes --save-db yes
```

Параметры:

* `--players` — количество игроков
* `--matches` — матчей на игрока
* `--save-csv` — сохранять ли в CSV
* `--save-db` — сохранять ли в БД
